GIT คืออะไร ทำงานอย่างไร และ แต่ละคำสั่งมีหน้าที่อย่างไรบ้าง
git คือเครื่องมือที่สามารถจัดการ version ของ code เอาไว้ใช้แก้ไข
เปลี่ยนแปลงลบ code คำสั่งเช่น git init :ใช้ในการเริ่มgit repository เริ่มแทรคในการทำงาน git clone ไว้ใช้ copy

Model View Controller (MVC) อธิบาย พร้อมยกตัวอย่าง 
MVC เป็นรูปแบบในการเขียนโค้ดรูปแบบหนึ่งที่ทำให้อ่านโค้ดได้ง่ายขึ้น
เป็นการแยกองค์ประกอบในแต่ละส่วน 
model จะมีข้อมูลที่เราเตรียมเอาไว้
view จะเป็นหน้าตาของส่วนทั้งหมดหรือเรียกว่า interface 
Controller จะเป็นศูนย์กลางทุกๆส่วนและเป็นโลจิก
ยกตัวอย่างในการทำงานเช่น เรารับข้อมูลมาจากลูกค้าหรือที่ View ก็จะติดต่อไป ที่
Controller เมื่อเราได้รับม้อมูลมาจาก view ก็จะไปติดต่อกับ Model เพื่อตรวจสอบความถูกต้องในส่วนนี้ในข้อมูลที่เรามี

3.จงอธิบายความสำคัญของการเขียน unit testing ความสำคัญของการเขียน unit testing คือ เราสามารถหาDebugcode ได้ง่ายขึ้นและสามารถพัฒนา Code ให้ได้อย่างมีประสิทธิภาพมากขึ้น

4.อธิบาย Continuous Integration / Continuous Deployment (CI/CD) เป็นระบบอัตโนมัติในการพัฒนาแอพลิเคชั่น CI คือทีมที่ทำงานในการแก้ไข
ปัญหาในการ merge โค้ดใหม่ๆโดยการTest ก่อนและส่งต่อไปให้ CD ในการพัฒนาและใช้งานระบบจริง

5.อธิบาย กฎหมาย Personal Data Protection Act (PDPA) และ บทลงโทษ

PDPA เป็นกฎหมายที่ตั้งขึ้นมาเพื่อคุ้มครองสิทธิส่วนบุคคลที่ผู้อื่นนำไปใช้แล้ว
ไม่ได้แจ้งให้ตนเองทราบ
กฎหมายเช่น PDPA มาตราที่ 27 ห้ามมิให้ผู้ควบคุมข้อมูลส่วนบุคคลใช้หรือเปิดเผย
ข้อมูลส่วนบุคคล โดยไม่ได้รับความยินยอมจากเจ้าของส่วนบุคคลมีโทษจำคุก
